# Server Utilization Agent

This project implements the MVP described in the PRD for an agent that:

- accepts a list of servers
- retrieves utilization metrics from Grafana or mock data
- flags servers below a `40%` utilization threshold
- estimates monthly and annual savings from removing underutilized servers
- writes Markdown and JSON reports
- includes a static UI that can be hosted on GitHub Pages

## What is included

- A Python CLI agent in [`src/server_utilization_agent`](/Users/ameyakulkarni/Desktop/Insurance/src/server_utilization_agent)
- A mock-data workflow so the agent can run immediately
- A Grafana Prometheus connector that uses Grafana's datasource query API
- CSV-based cost lookup for per-server monthly cost
- Unit tests for the analyzer and end-to-end mock flow
- A static dashboard UI in [`docs`](/Users/ameyakulkarni/Desktop/Insurance/docs)

## Quick start

Run the agent against the bundled mock data:

```bash
cd /Users/ameyakulkarni/Desktop/Insurance
PYTHONPATH=src python3 -m server_utilization_agent.cli \
  --config configs/example-config.mock.json \
  --servers-file configs/example-servers.txt \
  --print-report
```

The agent writes reports into [`outputs`](/Users/ameyakulkarni/Desktop/Insurance/outputs).

To generate a fresh report and inspect it in the UI, run the CLI and then upload the resulting JSON file from [`outputs`](/Users/ameyakulkarni/Desktop/Insurance/outputs) into the dashboard.

## Running against Grafana

1. Copy [`configs/example-config.grafana.json`](/Users/ameyakulkarni/Desktop/Insurance/configs/example-config.grafana.json) and update:
   - `base_url`
   - `datasource_uid`
   - PromQL expressions for `cpu` and `memory`
2. Export a Grafana service-account token:

```bash
export GRAFANA_API_TOKEN="your-token"
```

3. Run the CLI:

```bash
PYTHONPATH=src python3 -m server_utilization_agent.cli \
  --config configs/example-config.grafana.json \
  --servers app-server-01 app-server-02
```

## Config notes

- The Grafana connector expects each metric query to return a percentage time series.
- The agent calculates the utilization score as `max(avg CPU %, avg Memory %)`.
- Any server below the configured threshold is marked `Underutilized`.
- Costs come from a CSV file with `server_id`, `monthly_cost`, and optional `currency`.

## GitHub Pages deployment

The repo includes a GitHub Actions Pages workflow at [`deploy-pages.yml`](/Users/ameyakulkarni/Desktop/Insurance/.github/workflows/deploy-pages.yml). Once the project is pushed to GitHub:

1. Enable GitHub Pages in the repository and set the source to `GitHub Actions`.
2. Push to `main`.
3. The workflow deploys the contents of [`docs`](/Users/ameyakulkarni/Desktop/Insurance/docs).

Because GitHub Pages is static hosting, keep Grafana tokens and any sensitive infrastructure data out of the UI bundle. Upload generated report JSON files when you want to inspect live data.
